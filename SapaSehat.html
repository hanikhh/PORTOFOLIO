A User-Centered Design Strategy

My approach centered on three principles: Clarity, Trust, and Engagement. I developed a user persona, "Sasa", to guide every design decision.

Feature Deep Dive

Each feature was designed with a specific user problem in mind, focusing on clear and effective solutions.

Smart Medication Reminders

A dedicated module to simplify complex medication schedules and reduce missed doses.

Problem

Standard alarm apps are not tailored for medical needs.

Design Solution

The UI allows inputting dosage, frequency, and schedule type. A clear timeline view provides a daily overview, reducing ambiguity and empowering user control.

On-Demand Doctor Consultation

A seamless, chat-based interface to get professional medical advice quickly and conveniently.

Problem

Getting professional medical advice for minor issues is inconvenient.

Design Solution

Doctor profiles with ratings build trust. The flow from selection to payment to chat is frictionless, ensuring users get help fast.

The Community Hub

A moderated forum that acts as a safe space for users to share experiences and find peer support.

Problem

Users need peer support and a space to share experiences.

Design Solution

A simple, feed-style layout with topics, likes, and replies encourages positive interaction and makes it easy to find relevant information.

Conclusion & Personal Reflection

The SapaSehat project was a profound learning experience in designing for a domain where empathy and trust are paramount. The key takeaway was the importance of simplifying complexity.

The final design successfully integrates multiple essential health features into a single, cohesive, and non-intimidating application. It stands as a testament to how user-centric design can create tools that genuinely support user well-being.
